WHAT CHANGED VS recon/04
========================
1. /install dead-end killed
   - /install page deleted; route 307s straight to github.com.
   - All install CTAs (Hero, CTA, Onboarding step 1, Settings PR-reviews,
     Automations, RepoStatus) bypass our domain entirely.
   - install/callback retained + hardened: signed-out users' installation_id
     survives OAuth by sending them back to the callback URL itself.

2. /hire + /discover merged
   - /hire is the only sourcing app. /discover 307s to /hire.
   - DiscoverApp.standalone removed; component lives at
     src/components/discover/DiscoverApp.tsx.
   - Default mode: 'explore' when no seed; 'network' only when seed present
     or signed-in user's own username applied.
   - Public semantic map browses without auth.
   - "My Network" tab shows an inline gate (not a wall) for signed-out users,
     with a "Browse the map instead" fallback so they keep the value path.
   - Navbar "Discover" relabeled to "Hire" (testid retained for analytics).

3. Funnel ordering enforced (memory: feedback_value_first_funnel)
   - Landing: pitch + live diagram BEFORE any install/sign-in ask.
   - Repo pages: free tier renders for everyone; gate only the premium depth.
   - Hire: map browses BEFORE any "sign in to see networks" prompt.


REMAINING FRICTION POINTS (next candidates)
===========================================
F1. /hire/match for signed-out users
    Check whether it's a hard wall today; if so, give it a public preview
    (like ScorecardSamplePreview) so JD-match becomes the second value entry on /hire.

F2. /hire JD-match link is a tiny top-right text link
    First-class parallel input would let both "search by username" and "match by JD"
    be primary entries on /hire. Currently username search dominates visually.

F3. Post-Stripe /welcome screen
    After Stripe success, users are dumped back on /pricing. No "you're in,
    here's the next thing" moment. Should land on /welcome with the install CTA
    if not already installed, otherwise /onboarding step 2.

F4. First-time OAuth with no explicit return target
    Currently the default callback is "/" — for fresh users this should go to
    /onboarding instead, so they get the staged flow. Gate this on session.isFirstSignIn.

F5. /discover → /hire is a single 307; /hire/map → /discover → /hire is now
    a double-hop. Trivial cleanup: change /hire/map source to /hire directly
    in next.config.mjs.

F6. RepoStatus install message clarity
    "Install GitHub App to access this private repo" is good; "Install on this org"
    would be even clearer when the repo's org is known. Low priority.

F7. Sample previews still missing for diagram/ai-slop/security
    ScorecardSamplePreview is the only feature with a real public preview.
    Diagram, ai-slop, security still show empty paywalls. Replicate the pattern.
